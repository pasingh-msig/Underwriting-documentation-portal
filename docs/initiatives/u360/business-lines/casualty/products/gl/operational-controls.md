# GL - Operational Controls

## Control Overview
GL requires controlled access, traceable exception handling, and monitored integration points.

## Preventive Controls
| Control ID | Control Name | Description | Owner | Frequency | Evidence |
|---|---|---|---|---|---|
| PC-GL-01 | Role-Based Access | Only authorized casualty users can access GL functions | Support Owner | Continuous | Access configuration |

## Detective Controls
| Control ID | Control Name | Description | Owner | Frequency | Evidence |
|---|---|---|---|---|---|
| DC-GL-01 | Validation Failure Monitoring | Detect high volume of failed validations | Support Owner | Continuous | Monitoring dashboard |

## Logging and Audit Trail Controls
User actions, validation failures, exceptions, and workflow transitions are logged.
